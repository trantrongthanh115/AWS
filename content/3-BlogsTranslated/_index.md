During my internship, I translated and summarized 3 AWS technical blogs covering key topics in data engineering, cloud storage, and database management. Each blog was selected because it aligns directly with the technologies and services I worked with throughout the program.

---

### [Blog 1 - Building an Offline Feature Store on AWS: Reusing ML Data More Effectively](3.1-Blog1/)

This blog explores how to build an Offline Feature Store using Amazon SageMaker Unified Studio and SageMaker Catalog, enabling the storage, management, and sharing of ML features in a centralized environment. It covers key benefits including efficient feature reuse, centralized management, and data consistency, while detailing how Amazon S3 Tables and Apache Iceberg optimize the workflow for Data Science and ML Engineering teams.

---

### [Blog 2 - Automating Machine Learning Migration to Amazon SageMaker with LeapLogic](3.2-Blog2/)

This blog introduces LeapLogic by AWS and Impetus, an automation-led modernization suite designed to accelerate the migration of legacy Machine Learning systems to Amazon SageMaker. It details the 4-step migration process (Assessment, Automated Transformation, Testing & Validation, Deployment) and highlights how it helps businesses reduce transition time, operational costs, and deployment risks when modernizing AI/ML workloads.

---

### [Blog 3 - Building an Enterprise API Management Solution with Amazon API Gateway](3.3-Blog3/)

This blog summarizes Roger Zhang et al.'s guide on building an enterprise API management (APIM) solution using Amazon API Gateway, AWS Lambda, and Amazon DynamoDB. It breaks down the system's architecture into a management state (APIM Portal) and a runtime state (APIM Core), showing how enterprises can securely manage APIs at scale, implement robust security/governance policies, and seamlessly integrate downstream services and AI/LLM applications in a serverless paradigm.
